How it works

The Tracecord bot listens passively and stores everything in a queryable database.

01
Bot joins server

The Tracecord bot is added to a Discord server with read permissions and begins passively listening to all accessible channels.

02
Messages captured

Every message, edit, deletion, reaction, attachment, and voice event is captured in real-time and written to the database.

03
Backfill runs

Historical message backfill runs on all accessible channels, filling in the history before the bot's join date where permitted.

04
Dashboard access

Query, search, and analyze everything through the dashboard with advanced search, user profiles, and timeline tools.

Documentation

Everything you need to use Tracecord effectively. Find answers to common questions or explore the full feature reference.

Getting Started

Tracecord is a passive Discord message capture and analysis system. Once the bot is present in a server, all messages, edits, deletions, reactions, and voice events are recorded in real-time. The dashboard provides a web interface to search and analyze this data.

Note: Tracecord only captures messages from servers where the bot has been explicitly added. Direct messages and servers without the bot are not archived.

Data is stored indefinitely unless a removal request is submitted and approved. See the removal request page for details on how to remove your data.

Using the Dashboard

The dashboard is organized into four main tabs: Overview, Message Search, Users, and Timeline Reconstruction. The top-right header search lets you quickly find any user by ID or username.

The Overview tab shows aggregate stats and a per-server leaderboard. Select any server tab to view its specific stats and top users.

All tables support clicking on usernames to open a detailed profile card with full behavioral analysis.

Performance: Queries over very large date ranges may be slow on large archives. Use the date filters in message search to narrow results.

Message Search

The advanced search supports full-text matching combined with structured filters. All filters are combinable — leave any field blank to omit it.

# Example: find deleted messages from a user in a server after a date query: (empty — match all) from: 342819... (user ID) server: select from dropdown is: deleted ✓ after: 2025-12-01

The has pills filter for messages containing images, videos, or file attachments. The is pills filter for message state: deleted, edited, reply, or thread.

User Profiles

Click any username anywhere in the dashboard to open a profile card. The profile is scope-aware — you can toggle between a specific-server view and a cross-server global view.

Profile tabs include: Stats, Messages, Voice, Writing Style, Identity, Reactions, Interactions, and Connections. Each tab loads data on demand.

Interaction Graph: The interactions tab shows a force-directed graph of who this user has replied to or been replied by. Drag nodes to rearrange. Click a node to open that user's profile.

Timeline Reconstruction

Timeline Reconstruction shows the true chronological record of messages — including those that have since been deleted or edited. You can filter to show only deleted messages, only edited messages, or statistical activity spikes.

Optional filters for user ID, guild ID, and channel ID let you narrow the timeline to a specific conversation thread. The export button generates a JSON report of the current filtered view.

The dashboard is backed by a REST API. All endpoints are accessible at the server root.

Frequently Asked Questions

Can I see messages from servers I'm not in? ›
The dashboard shows messages from all servers the Tracecord bot is in. There is no per-user restriction on which servers you can query — it's a shared index.
How quickly are deleted messages captured? ›
Deletion events are captured in real-time via the Discord gateway. As long as the Tracecord bot is online when a message is deleted, it will be recorded. There is no way to retroactively delete a message from Tracecord once it's been captured.
Does the backfill capture messages before the bot joined? ›
Yes — the backfill process uses the Discord API to retrieve historical messages from accessible channels before the bot joined. However, it cannot recover already-deleted messages from before the bot's join date, as those are not returned by the API.
How do I remove my messages from Tracecord? ›
You can submit a removal request on the removal request page. Provide your Discord user ID and a reason. Requests are reviewed and processed within 7 days.
What is the "writing style fingerprint"? ›
Each user's writing style is analyzed for word length, sentence structure, punctuation usage, capitalization patterns, and common vocabulary. This produces a statistical signature that can be used to compare writing styles across users or detect alt accounts.

Data Removal

We respect data removal requests. To request removal of your messages, navigate to the removal request page and complete the form.

Important: Removal requests cover your messages in the Tracecord database. They do not affect what Discord stores or what other users may have screenshotted.

Removal requests are reviewed manually. You'll need to provide your Discord user ID and verify that you are the account owner. Processing typically takes up to 7 days.
